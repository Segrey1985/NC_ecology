Инструкция по подготовке архива

Для корректной работы системы PDF-файлы смежных разделов должны быть переименованы согласно Постановлению Правительства РФ № 87.

Шаг 1. Переименуйте PDF-файлы, добавив префикс раздела перед именем файла:

ПрефиксРазделПример имени файла
1_Пояснительная записка (ПЗ)1_Проект_ПЗ.pdf
2_Схема планировочной организации (ПЗУ)2_Проект_ПЗУ.pdf
3_Архитектурные решения (АР)3_Проект_АР.pdf
4_Конструктивные решения (КР)4_Проект_КР.pdf
5_Инженерные системы (ИОС)5_Проект_ИОС.pdf
5.1_Система электроснабжения5.1_Проект_ЭС.pdf
5.2_Система водоснабжения5.2_Проект_ВС.pdf
5.3_Система водоотведения5.3_Проект_ВО.pdf
5.4_ОВИК5.4_Проект_ОВИК.pdf
6.1_Технологические решения (ТМ)6.1_Проект_ТМ.pdf
7_Проект организации строительства (ПОС)7_Проект_ПОС.pdf
ИГИ_Инженерно-геологические изысканияИГИ_Отчет.pdf
ИЭИ_Инженерно-экологические изысканияИЭИ_Отчет.pdf

Шаг 2. Выделите все переименованные PDF-файлы → правый клик → «Добавить в архив» (ZIP или RAR).

Шаг 3. Загрузите полученный архив в форму на вкладке «Генерация документов».

Ограничения:

  • Максимальный размер архива: 200 МБ
  • Максимальный распакованный размер: 800 МБ
  • Максимальное количество файлов: 5000
  • Зашифрованные и многотомные архивы не поддерживаются
  • В архиве должен быть хотя бы один PDF-файл
  • Поддерживаются форматы архивов: ZIP и RAR

Что вы получите на выходе

После обработки система вернёт ZIP-архив с результатами. В зависимости от выбранной главы архив содержит:
  • DOCX-файлы — готовые главы раздела ООС, открываются в Microsoft Word
  • JSON-файлы — структурированные извлечённые данные для проверки
  • Логи — служебные записи о ходе генерации

Подсветка плейсхолдеров в DOCX:

  • Зелёным выделены автоматически заполненные значения.
  • Жёлтым выделены незаполненные поля, требующие ручного ввода.
  • Даты, колонтитулы и штамп заполняются автоматически.

Важно: Все сформированные документы требуют проверки и доработки специалистом. Система не заменяет эколога.

О системе NC_ecology

Программный инструмент для автоматизированной подготовки разделов экологической проектной документации (ПМООС / ООС).

Система использует подход RAG (Retrieval-Augmented Generation): сначала ищет релевантные фрагменты в загруженных PDF-документах, затем передаёт найденный контекст языковой модели для формирования структурированного ответа.

Технический стек

  • Backend: FastAPI + LangGraph (RAG-пайплайн)
  • Векторная БД: Qdrant (индексация PDF-фрагментов)
  • БД сессий: MongoDB
  • LLM: gpt-5-mini через OpenAI-совместимый API
  • Эмбеддинги: paraphrase-multilingual-MiniLM-L12-v2 (локально)
  • Reranker: cross-encoder/ms-marco-MiniLM-L-6-v2 (локально)
  • Архивы: поддержка ZIP и RAR
